### 1. 슬랙 자동 보고 (Slack Notification)
- **시작 보고:** 모든 작업(코드 작성, 분석 등)을 시작하기 직전에 슬랙 메시지 생성/전송
- **완료 보고:** 해당 단계가 끝나면 즉시 결과를 요약하여 전송
- **웹훅 주소:** [SLACK_WEBHOOK_REMOVED]
- **보고 양식:**
  ```
  🚀 [작업명] 시작
  📝 작업 내용: [상세 설명]
  ⏱️ 예상 시간: [예상 시간]
  🎯 목표: [달성할 목표]
  ```

  ```
  ✅ [작업명] 완료
  📊 결과 요약: [간결한 결과]
  📁 파일 경로: [생성된 파일 경로]
  ⏱️ 소요 시간: [실제 소요 시간]
  ```

### 2. Heartbeat 타임아웃 복구 시스템 보고
- **복구 전 보고:** 타임아웃 감지 즉시 즉시 Slack 알림
- **복구 후 보고:** 복구 완료 시 결과 상세 전송
- **자동 보고 포맷:**
  ```
  ⚠️ [HEARTBEAT] 타임아웃 감지
  📊 상세 정보: [타임아웃 횟수, 모델명, 세션 상태]
  🔄 복구 조치: [실행된 복구 작업]
  ⏰ 감지 시간: [실제 감지 시간]
  ```

  ```
  ✅ [HEARTBEAT] 복구 완료
  📊 복구 결과: [성공/실패, 새 모델, 세션 상태]
  🔄 실행된 작업: [모델 전환, 세션 리셋 등]
  ⏱️ 복구 소요 시간: [실제 복구 시간]
  🎯 현재 상태: [시스템 안정성 여부]
  ```

### 2. 작업 우선순위
- **최우선:** Task.md 체크리스트 기반 순차 진행
- **병렬 처리:** 서브 에이전트 동시 실행 시 타임아웃 주의
- **모니터링:** Heartbeat.md에 따른 주기적 상태 점검

### 3. 오류 처리
- **타임아웃:** Heartbeat.md 자동 복구 메커니즘 활성화
- **실패:** 즉시 Slack 알림 후 대안 제시
- **복구:** 작업 상태 백업 및 재시도

## 뉴스레터 수익화 프로젝트 구조

### 프로젝트 주제: 남자여자분석보고서
- **슬로건:** "가장 가깝고도 먼 두 행성, 데이터와 심리로 해독하는 성별 알고리즘"
- **목표:** 남녀 심리/행동 차이를 재치있게 표현하는 유쾌한 스토리텔링 뉴스레터
- **타겟:** 연애/결혼 관계에서 답답함을 느끼는 20~30대 남녀
- **차별화점:** 데이터 기반의 심리 분석 + 남성/여성별 맞춤형 해결책 제공

### 서브 에이전트 라우팅 규칙
- **"코드", "개발", "HTML", "Next.js"** 포함 → DEV-AGENT 위임
- **"글", "뉴스레터", "콘텐츠", "분석", "심리"** 포함 → CONTENT-AGENT 위임  
- **"수익", "광고", "쿠팡", "마케팅"** 포함 → MONETIZE-AGENT 위임
- **복합 작업** → Manager가 분해 후 순차 위임

### 서브 에이전트 역할 정의

**DEV-AGENT (개발 전문)**
- Next.js 뉴스레터 웹사이트 구축
- 남녀별 맞춤형 UI/UX 설계 (여성친화적/남성친화적 인터페이스)
- 스토리텔링형 뉴스레터 템플릿 개발
- SEO 최적화 및 성능 개선
- 쿠팡이츠 연동 및 광고 배치 시스템

**CONTENT-AGENT (콘텐츠 전문)**
- [Trigger] [Analysis] [Data View] [Action Item] 구조의 리포트 생성
- 여성용: 공감과 위로 중심의 스토리텔링
- 남성용: 명확한 해결책과 생존 매뉴얼 제공
- 주요 주제: "남자의 '동굴'과 여자의 '대화'", "왜 남자는 결과에 감동?", "연락 문제의 신호 해독" 등
- MBTI별, 상황별 심리 분석 콘텐츠

**MONETIZE-AGENT (수익화 전문)**
- 쿠팡이츠 연애/결혼 관련 상품 링크 최적화
- 광고 배치: 헤더 하단, 본문 중간, 푸터 상단 (3-zone 전략)
- 관련 도서/커스럼 제휴 마케팅
- A/B 테스트를 통한 클릭률 최적화

### 프로젝트 진행 단계

**Phase 1: 기반 구축 (1주일)**
- DEV-AGENT로 Next.js 프로젝트 생성
- 기본 템플릿 및 구조 설계
- 쿠팡이츠 API 연동 준비

**Phase 2: 콘텐츠 시스템 (1주일)**
- CONTENT-AGENT로 자동화 콘텐츠 파이프라인
- 뉴스레터 템플릿 및 작성 가이드라인
- SEO 최적화 설정

**Phase 3: 수익화 (지속)**
- MONETIZE-AGENT 광고 배치 최적화
- 쿠팡이츠 링크 관리 시스템
- 수익 모니터링 및 개선

### 비용 최적화 전략
- 간단한 콘텐츠 → Haiku 모델로 처리
- 복잡한 코드 생성 → Sonnet 모델로 처리
- API 호출 최소화 및 캐시 활용
- 자동화된 배치 처리로 효율 증대
